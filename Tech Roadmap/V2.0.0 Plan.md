## 二、P0：立即执行

### P0-1 — 5年财务趋势数据

**当前问题**

所有财务指标均只取当年单点值。单年ROE无法判断护城河强弱，单年净利率无法判断定价权趋势。更严重的是，毛利率和ROIC两个核心价值投资指标在当前系统中完全缺失。

**改造内容**

- 修改数据获取模块，AKShare 的 `ak.stock_financial_abstract` 支持多年拉取，改为获取最近5年（不足5年则退而取3年）
- 新增计算指标（当前完全缺失）：
  - **毛利率**：竞争护城河最直接体现，下行即定价权丧失的预警
  - **ROIC（投入资本回报率）**：比ROE更准确衡量资本效率，剔除杠杆影响
  - **营收5年CAGR**：成长股核心指标，复合增速才代表真实成长性
- 财务章节新增趋势表格，显示5年逐年值和方向趋势（↑ / ↓ / →）
- 5年均值进入估值模块，避免单年异常值污染DCF假设

**需要5年趋势的核心指标**

| 指标 | 必要性说明 | 历史要求 | 当前状态 |
|------|-----------|----------|----------|
| ROE | 持续ROE>15%才证明真实护城河，单年高值可能偶然 | 5年 | ❌ 仅当年 |
| 净利率 | 区分定价权提升 vs 成本压缩，趋势下行是预警 | 5年 | ❌ 仅当年 |
| 毛利率 | 竞争护城河最直接体现，下行=定价权丧失 | 5年 | ❌ 完全缺失 |
| 营收CAGR | 单年高增可能是低基数，5年复合才代表真实成长 | 5年 | ❌ 仅当年 |
| FCF转化率（FCF/净利） | 利润可被会计处理，现金不会说谎 | 5年 | ❌ 仅当年 |
| ROIC | 比ROE更准确，剔除杠杆影响 | 3年 | ❌ 完全缺失 |
| D/E负债率 | 看企业是否在悄悄加杠杆 | 3年 | ⚠️ 仅当年 |
| 研发费用率 | 科技/医药成长护城河可持续性 | 3年 | ❌ 缺失 |
| 应收账款周转天数 | 业务质量和客户议价能力的代理指标 | 3年 | ❌ 缺失 |
| 经营现金流 | 单独列出，不被净利润掩盖 | 3年 | ❌ 仅当年 |

> **不需要历史趋势的指标（点值即可）**：当前P/E、P/B、EV/EBITDA估值倍数（历史倍数参考价值有限）；流动比率（行业均值比历史趋势更有参考价值）；股息率（近2年已足够）。

**验收标准**

任意标的的财务章节出现5年ROE / 净利率 / 毛利率 / 营收 / FCF趋势表，ROIC出现在附录指标列表中。

### P0-2 — 数据溯源附录

**当前问题**

报告中每个数字的来源用户无法查证。数值是哪个年度的、通过哪个接口取的、是否经过处理，用户完全不知道，报告可信度存疑。

**改造内容**

- 在报告末尾新增「数据溯源表」，记录：数据项 / 数值 / 来源接口 / 取数时间 / 原始字段名
- 每个数据获取点加 metadata 记录（接口名 + 取数时间戳 + 原始字段）
- 新闻情绪章节的每条新闻保存来源 URL 和发布时间，报告内标注
- Tavily 搜索（P1-1上线后）的每个搜索结果 URL 同步写入溯源表

**溯源表格式示例**

| 数据项 | 数值 | 来源接口 | 取数时间 | 备注 |
|--------|------|----------|----------|------|
| 营收YoY | +21.8% | ak.stock_financial_abstract | 2025-09-30 | 最近一期年报 |
| DPS | ¥1.15 | ak.stock_history_dividend | 2025/2026年 | 两年平均 |
| 行业背景（新闻） | — | Tavily Search | 2026-03-10 | URL: https://... |

**验收标准**

任意标的的报告附录出现完整数据溯源表，所有关键财务指标和估值输入值均有对应来源记录。

## 三、P1：次优先

### P1-1 — 接入 Tavily Web Search（行业调研增强）

**当前问题**

Ch1行业背景和Ch2竞争力分析依赖LLM训练数据中的通用行业知识，缺乏实时市场份额数据、竞争对手近期动态和行业专项统计。分析结论难以落到具体数字，可信度不足。

**选型结论：Tavily API**

Tavily 是专为 LLM / AI Agent 设计的搜索 API，返回预清洗的结构化文本，直接可注入 Prompt，无需 HTML 解析。

- 免费额度：1000次/月，无需信用卡，可直接开始使用
- 每份报告搜索3次成本 ≈ $0.024，极低
- 与现有 LLM 完全解耦：Tavily 负责信息获取，现有 LLM 负责分析，架构不变

**Web Search 选型对比**

| 方案 | 特点 | 中文搜索质量 | 费用 | 推荐 |
|------|------|------------|------|------|
| **Tavily API** | 专为LLM设计，结构化输出，带来源URL | ★★★★ | 免费1000次/月，超出$0.008/次 | ✅ 首选 |
| OpenAI Web Search | GPT-4o原生，零额外依赖，一个参数开启 | ★★★ | ~$0.025/次，无免费额度 | 备选（成本较高） |

**改造内容**

- 在生成 Ch1 前：搜索「[公司名] 行业市场规模 [年份]」，注入行业背景 Prompt
- 在生成 Ch2 前：搜索「[公司名] 市场份额 竞争对手」+「[公司名] 近期动态」，注入竞争分析 Prompt
- 每次搜索结果的 URL 自动写入 P0-2 的数据溯源表
- Ch2 Prompt 新增强制约束：「必须包含：①主要竞争对手名称、②市场集中度CR3/CR5估算、③公司差异化竞争优势，不得用财务指标替代」

**集成示例**

```python
from tavily import TavilyClient

tavily = TavilyClient(api_key="tvly-...")

# 在生成 Ch2 前执行搜索
results = tavily.search(
    query=f"{company_name} 市场份额 竞争对手 {year}",
    max_results=5
)

# 将搜索结果注入 Prompt 上下文
context = "\n".join([r["content"] for r in results["results"]])
sources = [r["url"] for r in results["results"]]
# 用 context 生成 Ch2，用 sources 写入溯源表
```

**验收标准**

Ch2竞争力分析出现具体的市场份额数字或竞争对手对比数据，且来源 URL 可在溯源表中查证。

### P1-2 — 情绪模块稳定性改造

**当前问题**

多次运行同一标的，情绪章节内容和评分不稳定。根本原因有两层：LLM temperature 参数未明确设置导致随机性较高，以及评分计算由 LLM 主导而非规则计算。

**改造内容**

- LLM temperature 统一设为 **0.5**（情绪模块，平衡稳定性与自然表达）
- 评分计算改为规则主导：LLM 只做新闻分类（正面 / 负面 / 中性），评分由代码加权计算
- Prompt 加入约束：「严格基于以下新闻列表作出结论，不得引入列表之外的信息」

**验收标准**

同一标的同一天运行3次，情绪评分差异 ≤ 0.05，章节结论方向一致。

## 四、P2：中期

### P2-1 — 行业估值完整定位（分位数 + 代表性公司）

**当前问题**

估值章节给出了 PE、PB 等倍数，但用户不知道这些倍数在行业中处于什么位置。无参照系，高估低估无从判断。

**改造内容**

利用 AKShare `ak.stock_a_indicator_lg` 批量获取同行业A股估值指标，在估值章节新增「行业估值定位」模块，展示以下信息：

- **分位数定位**：当前标的 PE / PB 处于行业的百分位
- **代表性公司对比表**：选取行业内4类参照公司展示完整估值

| 分组 | 选取逻辑 | 代表数量 |
|------|---------|---------|
| 行业龙头（TOP 1-2） | 市值最大，估值溢价基准 | 2家 |
| 行业中位 | 最接近中位数PE的公司 | 1-2家 |
| 本标的 | 当前分析对象 | 1家 |
| 行业末尾 | 估值最低（可能是价值陷阱或低估）| 1-2家 |

**代表性公司对比表格式示例**

| 公司 | 市值(亿) | PE | PB | ROE | 股息率 | 备注 |
|------|---------|----|----|-----|--------|------|
| 行业龙头A | 5000 | 35x | 4.2x | 18% | 1.2% | TOP1 |
| 行业中位B | 800 | 28x | 2.8x | 12% | 2.1% | 中位 |
| **本标的** | 1200 | 45x | 5.1x | 14% | 0.8% | **当前分析** |
| 行业末尾C | 150 | 18x | 1.2x | 6% | 3.5% | 末尾 |

**验收标准**

任意标的的估值章节出现行业 PE / PB 分位数及代表性公司对比表，本标的的估值位置清晰可见。

### P2-2 — BUG-04 / BUG-05：综合建议与竞争分析完整性

**BUG-04：情绪结论未注入综合建议（Ch7）**

- 当前问题：Ch7 Prompt 只接收基本面 / 估值信号，情绪章节的结论完全未传入，综合建议逻辑不完整
- 改造：`ChapterContext` 新增 `sentiment_summary` 字段，将情绪评分 + 主要新闻观点摘要传入 Ch7 Prompt

**BUG-05：竞争分析用财务指标替代行业分析**

- 当前问题：Ch2竞争力分析充斥 ROE / 毛利率等财务指标描述，而非真实的竞争格局分析
- 改造：P1-1的 Tavily 搜索上线后，信息来源质量自然提升；同步在 Ch2 Prompt 中加入强制结构约束（见 P1-1 改造内容）

## 五、P3：长期规划

### P3-1 — 扩展行业覆盖：4个缺失行业

当前14个行业类型中，以下4个完全缺乏估值方法支持，无法生成有效报告：

| 行业 | 代表标的 | 主力估值方法 | 核心挑战 |
|------|---------|------------|---------|
| 医药（研发期） | 君实生物 688180.SH | rNPV法（药物管线折现） | 需要构建管线价值模型，AKShare无管线数据 |
| 周期材料 | 紫金矿业 601899.SH | 资产重置价值 + NAV | 金属价格周期性，DCF假设极不稳定 |
| 新能源 | 宁德时代 300750.SZ | PEG + EV/EBITDA | 补贴政策依赖，产能利用率是关键变量 |
| 房地产/REITs | 招商蛇口 001979.SZ | NAV折价率 + RNAV | 需要项目层面资产数据，免费接口无法覆盖 |

---

### P3-2 — 可比公司池横向对比

为每个行业建立5-10只可比公司池，每次分析时自动拉取可比公司的核心财务指标生成横向对比表格。这是 P2-1 分位数功能的深化——从「我在行业中处于几分位」升级到「我与具体竞争对手逐项对比」，对报告质量是质变级提升。

---

### P3-3 — 巨潮资讯年报PDF解析

公司自己披露的数据是最可信的原始来源。分业务收入拆分、产能数据、研发费用明细等信息，AKShare 免费接口无法覆盖，只能通过解析巨潮资讯的年报 PDF 获取（工具：`pdfplumber`）。工程量较大，作为长期数据质量提升方向。

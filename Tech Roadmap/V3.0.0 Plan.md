## 一、问题诊断：为什么当前架构失败

### 1.1 根本矛盾

当前架构的核心假设是「AKShare行业标签是可靠的、稳定的、可穷举的」。这三条假设全部不成立：

- **不可靠**：宁德时代被标记为医药生物，工业富联被标记为高端智能制造及工业互联网，这些字符串与YAML关键词无法匹配
- **不稳定**：AKShare的行业分类会随申万指数修订、监管口径变化而漂移，系统代码无法感知
- **不可穷举**：行业内部不断分化（SaaS vs 传统软件、创新药 vs 仿制药），加上跨界企业（比亚迪同时是电池+汽车+储能），规则数量会无限膨胀

### 1.2 失败的直接证据

```
# 本次测试中的WARN日志（7个标的中出现5次）

WARNING [Industry] Profile for '教育服务' not found, using generic
WARNING [Industry] Profile for '白酒' not found, using generic  ← 茅台也miss了
WARNING [Industry] Profile for '新能源' not found, using generic ← 宁德时代第一轮miss
```

**命中率统计**：7个标的中，直接命中profile的只有 insurance / real_estate / brand_moat（白酒通过关键词二次命中）。原始命中率约40%，其余全部fallback到generic 8x综合行业。

### 1.3 架构范式对比

| 维度 | 当前架构（查字典） | 新架构（看本质） |
|:-----|:-----------------|:---------------|
| 依赖 | AKShare行业字符串 → YAML关键词 | 财务数据特征 + LLM推理 |
| 覆盖率 | 受限于关键词列表，约40% | 硬规则探针70% + LLM兜底100% |
| 新行业应对 | 需要手动更新YAML | 自动（LLM天然up-to-date） |
| 跨界企业 | 无法处理 | LLM理解主营业务后动态判断 |
| 维护成本 | 高（每次变化改代码） | 低（只维护硬规则探针） |
| 可解释性 | 差（映射关系隐藏在配置里） | 好（LLM输出rationale字段） |

## 三、前置依赖：数据层专属科目提取

> **v3.1新增**：这是第一层探针能够正常工作的物理前提，必须在探针逻辑之前完成。

第一层探针依赖 `has_loan_loss_provision`、`has_insurance_reserve` 等布尔字段。如果数据清洗层没有提取这些科目，探针条件永远为False，第一道防线形同虚设。

### 3.1 在 `akshare_fetcher.py` 中新增科目扫描

```python
# akshare_fetcher.py

# 银行专属科目关键词（存在任一即视为银行报表结构）
BANK_BALANCE_SHEET_KEYWORDS = [
    '贷款和垫款', '发放贷款及垫款', '吸收存款', '向中央银行借款',
    '贷款损失准备', '应收款项类投资', '存放同业款项'
]

# 保险专属科目关键词
INSURANCE_BALANCE_SHEET_KEYWORDS = [
    '未到期责任准备金', '未决赔款准备金', '寿险责任准备金',
    '长期健康险责任准备金', '保户储金及投资款', '保费收入'
]

def extract_industry_flags(balance_sheet_df: pd.DataFrame) -> dict:
    """
    扫描资产负债表科目名称，提取行业专属标志位。
    
    这是第一层探针的数据基础，必须在metrics构建阶段完成。
    采用科目名称模糊匹配，而不是精确匹配，以应对不同年报的表述差异。
    """
    flags = {
        'has_loan_loss_provision': False,
        'has_insurance_reserve': False,
    }
    
    if balance_sheet_df is None or balance_sheet_df.empty:
        return flags
    
    # 提取所有科目名称（行名或列名，取决于数据格式）
    all_items = []
    if hasattr(balance_sheet_df, 'index'):
        all_items.extend(balance_sheet_df.index.tolist())
    if hasattr(balance_sheet_df, 'columns'):
        all_items.extend(balance_sheet_df.columns.tolist())
    
    all_items_str = ' '.join([str(item) for item in all_items])
    
    # 银行科目检测
    bank_hits = sum(
        1 for kw in BANK_BALANCE_SHEET_KEYWORDS 
        if kw in all_items_str
    )
    flags['has_loan_loss_provision'] = bank_hits >= 2  # 至少2个科目命中才判定
    
    # 保险科目检测
    insurance_hits = sum(
        1 for kw in INSURANCE_BALANCE_SHEET_KEYWORDS 
        if kw in all_items_str
    )
    flags['has_insurance_reserve'] = insurance_hits >= 2
    
    return flags

def build_metrics(stock_code: str, financial_data: dict) -> dict:
    """
    在构建metrics时，调用科目扫描，将标志位注入metrics字典。
    """
    metrics = {}
    
    # ... 现有的metrics构建逻辑 ...
    
    # 新增：注入行业专属标志位
    balance_sheet = financial_data.get('balance_sheet')
    industry_flags = extract_industry_flags(balance_sheet)
    metrics.update(industry_flags)
    
    return metrics
```

## 四、第一层详细设计：财务特征硬规则探针

### 4.1 设计原则

探针识别的是**商业模式本质**，而不是行业标签。每个规则对应一个在财务报表上有独特指纹的商业模型。同时引入**时间维度平滑**，防止单年异常值导致误判：困境企业判定使用3年均值，护城河判定使用5年均值。

### 4.2 完整实现

```python
from dataclasses import dataclass, field
from typing import Optional

@dataclass
class SpecialRegimeResult:
    regime: str
    confidence: float
    triggered_rules: list[str] = field(default_factory=list)

SPECIAL_REGIME_CONFIGS = {
    'bank': {
        'primary_methods': ['pb_roe', 'ddm'],
        'weights': {'pb_roe': 0.6, 'ddm': 0.4},
        'disabled_methods': ['ev_ebitda', 'graham_number', 'dcf'],
        'exempt_scoring_metrics': ['debt_equity', 'current_ratio', 'fcf_ni'],
        'scoring_mode': 'financial',
        'ev_ebitda_multiple_range': (0.0, 0.0),  # 不适用
    },
    'insurance': {
        'primary_methods': ['pb_roe', 'ddm', 'pe'],
        'weights': {'pb_roe': 0.4, 'ddm': 0.35, 'pe': 0.25},
        'disabled_methods': ['ev_ebitda', 'graham_number'],
        'exempt_scoring_metrics': ['debt_equity', 'current_ratio', 'fcf_ni'],
        'scoring_mode': 'financial',
        'ev_ebitda_multiple_range': (0.0, 0.0),
    },
    'real_estate': {
        'primary_methods': ['pb'],
        'weights': {'pb': 1.0},
        'disabled_methods': ['graham_number', 'ev_ebitda', 'dcf'],
        'exempt_scoring_metrics': [],
        'pb_multiple_cap': 0.5,
        'scoring_mode': 'standard',
        'ev_ebitda_multiple_range': (0.0, 0.0),

    },
    'distressed': {
        'primary_methods': ['ev_sales', 'net_net', 'asset_replacement'],

        'weights': {'pe_moat': 0.5, 'dcf': 0.3, 'ev_ebitda': 0.2},
        'disabled_methods': ['graham_number'],
        'exempt_scoring_metrics': [],
        'scoring_mode': 'standard',
        'ev_ebitda_multiple_range': (15.0, 25.0),
    },
    'pharma_innovative': {
        'primary_methods': ['ps', 'ev_sales'],
        'weights': {'ps': 0.6, 'ev_sales': 0.4},
        'disabled_methods': ['pe', 'graham_number', 'ev_ebitda'],
        'exempt_scoring_metrics': ['fcf_ni', 'net_margin'],
        'scoring_mode': 'standard',
        'ev_ebitda_multiple_range': (0.0, 0.0),
    },
}

def detect_special_regime(
    metrics: dict,
    company_info: dict
) -> Optional[SpecialRegimeResult]:
    """
    第一层：基于财务特征的硬规则探针。
    
    完全不依赖行业标签字符串。返回None表示未命中，交由第二层处理。
    
    前置条件：metrics中必须包含 extract_industry_flags() 提取的标志位。
    若标志位缺失（has_loan_loss_provision等），银行/保险规则将自动跳过（不误判）。
    
    原则：宁可miss也不要误判（false positive比false negative代价更高）。
    """
    
    # ── 规则1：银行 ─────────────────────────────────────────────
    # 指纹：超高杠杆（存贷款结构天然如此）+ 贷款损失准备金科目
    # 注意：has_loan_loss_provision由数据层extract_industry_flags()提供
    de_ratio = metrics.get('de_ratio', 0) or 0
    has_provision = metrics.get('has_loan_loss_provision', False)

    if de_ratio > 8 and has_provision:
        return SpecialRegimeResult(
            regime='bank',
            confidence=0.95,
            triggered_rules=[f'de_ratio={de_ratio:.1f}x > 8', 'has_loan_loss_provision=True']
        )

    # ── 规则2：保险 ─────────────────────────────────────────────
    # 指纹：高杠杆（但低于银行）+ 保险准备金科目
    has_insurance_reserve = metrics.get('has_insurance_reserve', False)

    if de_ratio > 4 and has_insurance_reserve:
        return SpecialRegimeResult(
            regime='insurance',
            confidence=0.92,
            triggered_rules=[f'de_ratio={de_ratio:.1f}x > 4', 'has_insurance_reserve=True']
        )

    # ── 规则3：房地产开发 ────────────────────────────────────────
    # 指纹：存货占总资产超40% + 大额预收款项（预售回款）
    total_assets = metrics.get('total_assets', 1) or 1
    inventory = metrics.get('inventory', 0) or 0
    advance_receipts = metrics.get('advance_receipts', 0) or 0

    inventory_ratio = inventory / total_assets
    has_large_advance = advance_receipts > total_assets * 0.10

    if inventory_ratio > 0.40 and has_large_advance:
        return SpecialRegimeResult(
            regime='real_estate',
            confidence=0.90,
            triggered_rules=[
                f'inventory_ratio={inventory_ratio:.1%} > 40%',
                f'advance_receipts占比={advance_receipts/total_assets:.1%} > 10%'
            ]
        )

    # ── 规则4：困境企业 ──────────────────────────────────────────
    # ⚠️ 使用3年均值而非单年，防止一次性商誉减值/资产处置误伤优质公司
    # 必须满足3条信号中的至少2条
    distress_signals = []

    net_margin_3yr = metrics.get('net_margin_3yr_avg') or metrics.get('net_margin', 0) or 0
    roe_3yr = metrics.get('roe_3yr_avg') or metrics.get('roe', 0) or 0
    consecutive_loss = metrics.get('consecutive_loss_years', 0) or 0

    if net_margin_3yr < -10:
        distress_signals.append(f'net_margin_3yr_avg={net_margin_3yr:.1f}% < -10%')
    if roe_3yr < -10:
        distress_signals.append(f'roe_3yr_avg={roe_3yr:.1f}% < -10%')
    if consecutive_loss >= 2:
        distress_signals.append(f'consecutive_loss_years={consecutive_loss} >= 2')

    # 排除银行/保险：高杠杆不代表困境
    is_financial = has_provision or has_insurance_reserve
    if len(distress_signals) >= 2 and not is_financial:
        return SpecialRegimeResult(
            regime='distressed',
            confidence=0.85,
            triggered_rules=distress_signals
        )

    # ── 规则5：消费护城河（高确信度版本）────────────────────────────
    # ⚠️ 阈值刻意设高（毛利70%+、ROE5yr 18%+、FCF4年+），宁可漏判不误判
    gross_margin = metrics.get('gross_margin', 0) or 0
    roe_5yr = metrics.get('roe_5yr_avg') or metrics.get('roe', 0) or 0
    fcf_positive_years = metrics.get('fcf_positive_years', 0) or 0

    if gross_margin > 70 and roe_5yr > 18 and fcf_positive_years >= 4:
        return SpecialRegimeResult(
            regime='brand_moat',
            confidence=0.88,
            triggered_rules=[
                f'gross_margin={gross_margin:.1f}% > 70%',
                f'roe_5yr_avg={roe_5yr:.1f}% > 18%',
                f'fcf_positive_years={fcf_positive_years} >= 4'
            ]
        )

    # ── 规则6：创新药研发 ─────────────────────────────────────────
    # 指纹：高研发费用率 + 亏损/微利 + 业务描述含管线关键词
    rd_ratio = metrics.get('rd_expense_ratio', 0) or 0
    net_margin = metrics.get('net_margin', 0) or 0
    business_desc = company_info.get('business_description', '')

    pipeline_keywords = ['临床', '管线', 'pipeline', 'IND', 'NDA', '一期', '二期', '三期', 'Phase']
    has_pipeline = any(kw.lower() in business_desc.lower() for kw in pipeline_keywords)

    if rd_ratio > 30 and net_margin < 5 and has_pipeline:
        return SpecialRegimeResult(
            regime='pharma_innovative',
            confidence=0.82,
            triggered_rules=[
                f'rd_ratio={rd_ratio:.1f}% > 30%',
                f'net_margin={net_margin:.1f}% < 5%',
                'has_pipeline_keywords=True'
            ]
        )

    return None  # 未命中任何规则，交给第二层
```

```python
import json

def extract_json_from_llm_output(text: str) -> dict:
    """
    从LLM输出中提取JSON，处理以下场景：
    1. DeepSeek-Reasoner的 <think>...</think> 思维链块
    2. ```json ... ``` markdown代码块
    3. ``` ... ``` 无语言标注的代码块
    4. 裸JSON（无任何包裹）
    
    提取顺序：优先取代码块（精确），其次取最大花括号对（兜底）。
    """
    
    # Step 1：移除 DeepSeek-Reasoner 的思维链块
    # <think> 块可能包含大量分析文字，必须先清除，否则后续正则可能匹配到错误内容
    text = re.sub(r'<think>.*?</think>', '', text, flags=re.DOTALL).strip()
    
    # Step 2：尝试提取 markdown 代码块（```json 或 ```）
    # 使用非贪婪匹配，防止跨越多个代码块
    code_block_pattern = r'```(?:json)?\s*(\{.*?\})\s*```'
    match = re.search(code_block_pattern, text, re.DOTALL)
    if match:
        return json.loads(match.group(1))
    
    # Step 3：尝试直接提取最外层的花括号对象
    # re.DOTALL 使 . 匹配换行符
    brace_pattern = r'\{.*\}'
    match = re.search(brace_pattern, text, re.DOTALL)
    if match:
        return json.loads(match.group(0))
    
    raise ValueError(f"无法从LLM输出中提取JSON。原始输出（前500字符）：{text[:500]}")
```

### 5.3 Pydantic Schema校验（含自动权重归一化）

> **v3.1改进**：将`weights`的校验从「严格验证加总=1」改为「自动归一化」，同时新增`method_importance`字段接收LLM的相对分数。

```python
from pydantic import BaseModel, validator, Field
from typing import Literal

ALLOWED_METHODS = {
    'pe', 'ev_ebitda', 'dcf', 'ps', 'pb_roe', 'ddm',
    'peg', 'normalized_pe', 'pe_moat', 'ev_sales',
    'asset_replacement', 'net_net'
}

ALLOWED_SCORING_MODES = {'standard', 'cycle_adjusted', 'financial', 'distressed'}

class ValuationConfig(BaseModel):
    regime: str
    primary_methods: list[str]
    
    # LLM提供相对重要性分数（1-10整数），Python层自动归一化为权重
    # 最终存储的是归一化后的权重（加总=1.0）
    weights: dict[str, float] = {}
    method_importance: dict[str, int] = {}  # LLM填写这个字段
    
    disabled_methods: list[str] = []
    exempt_scoring_metrics: list[str] = []
    scoring_mode: str = 'standard'
    ev_ebitda_multiple_range: tuple[float, float] = (8.0, 12.0)
    rationale: str = ''
    confidence: float = Field(ge=0.0, le=1.0, default=0.7)
    source: Literal['hard_rule', 'llm', 'fallback'] = 'llm'
    
    @validator('primary_methods')
    def methods_must_be_allowed(cls, v):
        invalid = set(v) - ALLOWED_METHODS
        if invalid:
            raise ValueError(f'非法估值方法: {invalid}，请从允许列表中选择')
        if not v:
            raise ValueError('至少需要一个估值方法')
        return v
    
    @validator('disabled_methods')
    def disabled_must_be_allowed(cls, v):
        invalid = set(v) - ALLOWED_METHODS
        if invalid:
            raise ValueError(f'disabled_methods包含未知方法: {invalid}')
        return v
    
    @validator('scoring_mode')
    def mode_must_be_valid(cls, v):
        if v not in ALLOWED_SCORING_MODES:
            raise ValueError(f'非法评分模式: {v}，允许值: {ALLOWED_SCORING_MODES}')
        return v
    
    @validator('weights', always=True)
    def auto_normalize_weights(cls, v, values):
        """
        ⚠️ v3.1核心改动：自动归一化，而不是严格校验加总=1。
        
        处理逻辑：
        1. 若weights为空但method_importance有值，从importance计算weights
        2. 若weights有值，直接归一化（容忍LLM的浮点误差）
        3. 若两者都为空，从primary_methods均分权重
        """
        importance = values.get('method_importance', {})
        methods = values.get('primary_methods', [])
        
        if not v and importance:
            # 从LLM的重要性分数计算权重
            v = {k: float(importance[k]) for k in methods if k in importance}
        
        if not v and methods:
            # 均分兜底
            v = {m: 1.0 for m in methods}
        
        if not v:
            return {}
        
        total = sum(v.values())
        if total == 0:
            raise ValueError('所有权重不能都为0')
        
        # 归一化，保留4位小数
        normalized = {k: round(val / total, 4) for k, val in v.items()}
        
        # 修正浮点累积误差（确保加总精确等于1.0）
        keys = list(normalized.keys())
        normalized[keys[-1]] = round(1.0 - sum(list(normalized.values())[:-1]), 4)
        
        return normalized
```

## 六、缓存策略

### 6.1 Cache Key设计

> **v3.1关键修正**：Cache Key从 `{fiscal_year}` 改为 `{report_period}`，以报告期（季度）为粒度，避免同一财年内不同季度共享缓存导致的「反应迟钝」问题。

```python
def get_cache_key(stock_code: str, metrics: dict) -> str:
    """
    缓存粒度：每支股票每个报告期一次LLM调用。
    
    v3.1修正：
    - 旧Key：valuation_config:{stock_code}:{fiscal_year}
      问题：2026全年共享同一缓存，Q1判定的框架在Q3仍然有效
    - 新Key：valuation_config:{stock_code}:{report_period}
      report_period示例：2025-12-31（年报）、2025-09-30（三季报）
    
    企业的估值逻辑属于慢变量，一个报告期内的框架判断是稳定的。
    唯一需要强制刷新的是重大重组事件（通过force_refresh参数触发）。
    """
    # 优先使用精确报告期，兜底使用财年
    report_period = (
        metrics.get('report_period')          # 优先：2025-09-30
        or metrics.get('fiscal_year_end')     # 次优：2025-12-31
        or metrics.get('fiscal_year')         # 兜底：2025
        or 'unknown'
    )
    return f"valuation_config:{stock_code}:{report_period}"
```

### 6.2 统一入口函数

```python
def get_valuation_config(
    stock_code: str,
    company_info: dict,
    metrics: dict,
    force_refresh: bool = False,
) -> ValuationConfig:
    """
    三层漏斗的统一入口。
    
    force_refresh=True 用于重大资产重组、借壳上市等场景，
    强制绕过缓存重新进行LLM推理。
    """
    
    # ── 第一层：硬规则探针（零成本） ─────────────────────────────
    regime_result = detect_special_regime(metrics, company_info)
    if regime_result:
        preset = SPECIAL_REGIME_CONFIGS[regime_result.regime].copy()
        config = ValuationConfig(
            regime=regime_result.regime,
            confidence=regime_result.confidence,
            source='hard_rule',
            rationale=f"财务特征探针命中: {', '.join(regime_result.triggered_rules)}",
            **preset
        )
        logger.info(
            f"[{stock_code}] 第一层探针命中: {regime_result.regime} "
            f"(confidence={regime_result.confidence:.2f})"
        )
        return config
    
    # ── 第二层：检查缓存 ─────────────────────────────────────────
    cache_key = get_cache_key(stock_code, metrics)
    
    if not force_refresh:
        try:
            cached = storage.get(cache_key)
            if cached:
                config = ValuationConfig(**json.loads(cached.value))
                logger.info(
                    f"[{stock_code}] 缓存命中: {config.regime} "
                    f"(period={metrics.get('report_period', 'unknown')})"
                )
                return config
        except Exception as e:
            logger.warning(f"[{stock_code}] 缓存读取失败（继续LLM推理）: {e}")
    
    # ── 第二层：LLM推理 ──────────────────────────────────────────
    try:
        config = call_llm_for_valuation_config(company_info, metrics)
        
        try:
            storage.set(cache_key, json.dumps(config.dict()))
            logger.debug(f"[{stock_code}] 估值配置已缓存: key={cache_key}")
        except Exception as e:
            logger.warning(f"[{stock_code}] 缓存写入失败（不影响本次结果）: {e}")
        
        return config
        
    except Exception as e:
        logger.error(
            f"[{stock_code}] LLM推理失败，降级到generic兜底: {e}"
        )
    
    # ── 第三层：安全兜底 ─────────────────────────────────────────
    logger.warning(f"[{stock_code}] 使用generic兜底配置，报告置信度将标注为低")
    return ValuationConfig(
        regime='generic',
        primary_methods=['ev_ebitda', 'dcf', 'pb'],
        weights={'ev_ebitda': 0.4, 'dcf': 0.4, 'pb': 0.2},
        disabled_methods=[],
        exempt_scoring_metrics=[],
        scoring_mode='standard',
        ev_ebitda_multiple_range=(7.0, 10.0),
        confidence=0.3,
        source='fallback',
        rationale='行业识别失败（LLM不可用），使用通用框架，结论仅供参考'
    )


def invalidate_config_cache(stock_code: str, reason: str):
    """
    事件驱动的缓存失效。
    重大资产重组、借壳上市、主营业务变更时调用，
    强制下次报告重新进行LLM推理。
    """
    try:
        keys_result = storage.list(prefix=f"valuation_config:{stock_code}:")
        deleted_count = 0
        for key in (keys_result.keys or []):
            storage.delete(key)
            deleted_count += 1
        logger.info(
            f"[{stock_code}] 估值配置缓存已清除 "
            f"({deleted_count}条记录)，原因: {reason}"
        )
    except Exception as e:
        logger.error(f"[{stock_code}] 缓存清除失败: {e}")
```
